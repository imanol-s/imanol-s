## Overview

A topological sorting-based solution for project scheduling using directed acyclic graphs (DAGs). Implements PERT/CPM methodologies to calculate:

- Earliest start/finish times (forward pass)
- Latest start/finish times (backward pass)
- Total float/slack time

For more reading: <a href="https://en.wikipedia.org/wiki/Program_evaluation_and_review_technique">PERT</a>

## ❓ Problem

1. Task dependencies created non-trivial scheduling constraints
2. Manual critical path identification proved error-prone for large projects (n > 100 tasks)
3. Schedule not optimally formatted in relation to time efficiency.

---

## 🎯 Goal

1. Develop O(V+E) algorithm for critical path identification
2. Model task dependencies in a DAG with cycle detection
3. Compute per-task scheduling windows and identify zero-slack critical tasks

---

## ✨ Solution

### Graph Representation

- Adjacency list structure with directed edges representing task dependencies
- DFS-based topological sort with `onStack[]` tracking to detect cycles — returns `null` if the graph is not a valid DAG

### PERT Implementation

- **Forward pass**: iterates vertices in topological order; for each vertex, `ES = max(EF of predecessors)`, `EF = ES + duration`
- **Backward pass**: initializes all vertices' `LF` to project completion time, then processes in reverse topological order; `LF = min(LS of successors)`, `LS = LF - duration`
- **Slack**: computed as `LF − EF` across all vertices using a parallel stream

### Critical Path Analysis

- Zero-slack tasks constitute the critical path
- `numCritical()` counts critical vertices; `criticalPath()` returns the total project duration (max EF across all vertices)
- Scheduling output per task: EC, LC, slack, and critical flag
